## Project Overview

Virtual Company Platform - A multi-company management dashboard where a CEO can manage multiple virtual companies, each with their own workflows and agent-based task execution.

## Build and Run Commands

```bash
# Start both services (recommended)
./start.sh              # Interactive startup
./start.sh --force      # Force restart, kill existing processes

# Backend (Go)
cd backend
go build -o bin/server ./cmd/server   # Build
go run cmd/server/main.go             # Run dev
make build                            # Alternative build

# Frontend (React)
cd frontend
npm install                           # Install dependencies
npm run dev                           # Run dev server (port 8100)
npm run build                         # Build for production

# Clear test data
rm -rf backend/data/companys/*
```

## Ports

- Backend API: 8181
- Frontend Dev: 8100

## Architecture

### Backend (Go + Aura Agent SDK)

**Core Flow**: Company → Session → Workflow → Steps → Agent Execution

```
Company (company.jsonl)
  └── Sessions (sessions/<id>.jsonl)
       ├── Workflow (generated by LLM or template)
       ├── Instances (role agent instances)
       └── Outputs (markdown files per step)
```

**Key Modules** (`backend/internal/`):

| Module | Purpose |
|--------|---------|
| `api/` | HTTP handlers, WebSocket for real-time progress |
| `task/engine.go` | Workflow execution engine - `RunWorkflow()` loop, `DecomposeTask()` for LLM generation |
| `agent/executor.go` | Aura SDK integration - `ExecuteInstance()`, `ExecutePlain()` |
| `workflow/` | DAG structure, step dependencies, status tracking |
| `role/` | Role definitions (pm, dev, qa, techlead, ceo), instance management |
| `session/` | Session lifecycle, status states |
| `company/` | Company model and store |

**Session Status Flow**:
```
planning → draft → approved → running → [paused → running] → completed/failed
```

**LLM Integration**: Uses Aura Agent SDK (local replace directives in go.mod point to `/Users/oneliang/AgentProjects/aura/`). Configure via `backend/configs/config.yaml`.

**Decision Points**: Steps marked `is_decision_point: true` pause workflow for CEO approval via `SubmitDecision` API. Resume with approve/continue/restart.

### Frontend (React + TypeScript + Vite + TailwindCSS)

**Key Components** (`frontend/src/components/`):

| Component | Purpose |
|-----------|---------|
| `CompanyList.tsx` | CEO dashboard - company cards with session stats |
| `CompanyDetailPage.tsx` | Session list per company, WebSocket for planning status |
| `SessionDetailPage.tsx` | Workflow topology, status display, WebSocket progress |
| `WorkflowTopology.tsx` | React Flow DAG visualization with status colors |
| `SessionOutputs.tsx` | Agent output documents (markdown files) |

**WebSocket Events** (backend → frontend):
- `step_start`: Step execution begins (frontend updates to yellow "进行中")
- `step_complete`: Step finished, triggers output refresh
- `workflow_generated`: LLM finished generating workflow
- `workflow_status`: Session status change

**Real-time State Updates**: Frontend locally updates workflow step status on WebSocket events before fetching from API (for immediate visual feedback).

### Data Storage (JSONL)

File-based storage in `backend/data/companys/<company_id>/`:
- `company.jsonl` - Company metadata
- `sessions/<id>.jsonl` - Session with workflow state
- `sessions/<id>/instances/<id>.jsonl` - Role instance execution history
- `sessions/<id>/outputs/step-<id>.md` - Agent output per step

### Role Configuration

Roles defined in `backend/configs/roles/*.yaml`:
- `pm.yaml` - Product Manager
- `dev.yaml` - Developer
- `qa.yaml` - QA Engineer
- `techlead.yaml` - Tech Lead

Each role has: `id`, `name`, `description`, `system_prompt`, `tools_allowed`.

### Workflow Template

`backend/configs/workflow_template.yaml` defines fallback workflow when LLM generation fails or executor unavailable.

## API Endpoints

### Company Management
- `POST /api/companies` - Create company
- `GET /api/companies?owner_id={id}` - List companies for CEO
- `GET /api/companies/{id}` - Get company detail
- `DELETE /api/companies/{id}` - Delete company

### Session Management
- `POST /api/companies/{id}/sessions` - Create session (async LLM workflow generation)
- `GET /api/companies/{id}/sessions` - List sessions
- `GET /api/companies/{id}/sessions/{sid}` - Get session with workflow
- `POST /api/companies/{id}/sessions/{sid}/approve` - Approve workflow
- `POST /api/companies/{id}/sessions/{sid}/start` - Start execution
- `POST /api/companies/{id}/sessions/{sid}/resume` - Resume from paused/failed
- `POST /api/companies/{id}/sessions/{sid}/decision` - CEO decision on decision point

### WebSocket
- `GET /ws?session_id={id}` - Session detail page connection
- `GET /ws?company_id={id}` - Company list page connection
